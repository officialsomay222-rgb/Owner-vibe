## 2024-05-16 - Reduce App Lag via Hardware Acceleration
**Learning:** Heavy CSS properties like `backdrop-blur` and large `blur` filters on `absolute`/`fixed` positioned elements force constant repaints on the main thread (CPU), resulting in significant UI lag on lower-end mobile devices and WebViews.
**Action:** Always append GPU acceleration utilities (`transform-gpu will-change-transform`) to computationally expensive visual elements (blurs, fixed headers, nav bars) to offload rendering to the device's GPU, immediately dropping latency without altering visual fidelity.
## 2024-05-23 - Prevent Global Event Churn with useLocalStorage
**Learning:** Passing inline objects/arrays (e.g., `[]`) as `initialValue` to custom hooks like `useLocalStorage` breaks referential equality on every render. If this parameter is included in a `useEffect` dependency array, it causes constant unmounting and remounting of global window event listeners (like 'storage' or 'local-storage'). Furthermore, dispatching a global 'local-storage' event without a specific payload forces every instance of the hook (e.g., ~20 instances across the app for settings, history, etc.) to unnecessarily parse JSON and evaluate React state.
**Action:** Wrap stable parameters like `initialValue` in a `useRef` to remove them from `useEffect` dependencies, preventing event listener churn. Always include a specific `key` in custom global events (`new CustomEvent('local-storage', { detail: { key } })`) and check it inside the listener (`if (event.detail.key !== hookKey) return;`) to early-exit and avoid redundant O(N) operations across all hook instances.
