## 2024-05-24 - React Context Re-render Optimization\n**Learning:** The `MusicContext` provides a lot of values including `currentTime` which updates very frequently (multiple times per second) during audio playback. Any component using `useMusic()` will re-render every time `currentTime` changes, even if it doesn't use `currentTime`. This causes unnecessary re-renders of heavy components like `HomeTab`, `SearchTab`, and `LibraryTab` in `App.tsx`.\n**Action:** Optimize contexts or component composition to prevent unrelated state updates from causing heavy view re-renders.
