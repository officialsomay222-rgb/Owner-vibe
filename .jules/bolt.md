## 2024-05-16 - Reduce App Lag via Hardware Acceleration
**Learning:** Heavy CSS properties like `backdrop-blur` and large `blur` filters on `absolute`/`fixed` positioned elements force constant repaints on the main thread (CPU), resulting in significant UI lag on lower-end mobile devices and WebViews.
**Action:** Always append GPU acceleration utilities (`transform-gpu will-change-transform`) to computationally expensive visual elements (blurs, fixed headers, nav bars) to offload rendering to the device's GPU, immediately dropping latency without altering visual fidelity.## 2024-05-18 - [Fix `useLocalStorage` causing application-wide re-renders]
**Learning:** Passing inline arrays/objects as `initialValue` to generic local storage hooks without memoization causes infinite effect loops and global re-render avalanches across all components using the hook due to the `storage` event broadcast pattern.
**Action:** Always wrap `initialValue` inside a `useRef` and dispatch a `CustomEvent` filtered by the target `key` to surgically trigger updates only in related components, drastically reducing UI thread blocks.
