## 2024-05-16 - useLocalStorage Global Event Re-render Bottleneck
**Learning:** `useLocalStorage` hooks that rely on global event listeners (like a single `local-storage` event) without key-specific filtering cause severe performance bottlenecks in React applications. In this app, many core components and hooks (like `searchHistory`, `favorites`, `offline_vault`, and multiple settings) used `useLocalStorage`. Updating any single setting triggered a global `local-storage` event, which woke up *every* component using `useLocalStorage` and caused them to parse JSON and update state, leading to massive unnecessary application-wide re-renders. Additionally, passing inline arrays/objects (like `useLocalStorage('key', [])`) as `initialValue` and including it in the `useEffect` dependency array caused infinite loops or excessive effect executions on every render.
**Action:** When implementing global state synchronization via `localStorage` events, always dispatch a `CustomEvent` that includes the updated `key` in its detail payload. Listeners must early-return if the event's key doesn't match their specific key. Furthermore, use deep equality checks (`JSON.stringify(prev) === JSON.stringify(newValue)`) before updating local React state to prevent re-renders when the actual value hasn't changed. Always exclude reference-type `initialValue` props from `useEffect` dependency arrays when they are only used as fallback defaults.
