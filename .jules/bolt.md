## 2024-05-16 - Reduce App Lag via Hardware Acceleration
**Learning:** Heavy CSS properties like `backdrop-blur` and large `blur` filters on `absolute`/`fixed` positioned elements force constant repaints on the main thread (CPU), resulting in significant UI lag on lower-end mobile devices and WebViews.
**Action:** Always append GPU acceleration utilities (`transform-gpu will-change-transform`) to computationally expensive visual elements (blurs, fixed headers, nav bars) to offload rendering to the device's GPU, immediately dropping latency without altering visual fidelity.
## $(date +%Y-%m-%d) - LocalStorage Hook Re-render Optimization
**Learning:** In a React application heavily utilizing `localStorage` across many components (like settings, history, and playlists), broadcasting a generic `Event("local-storage")` for *any* key change forces *every* component using the `useLocalStorage` hook to wake up, read from localStorage, parse the JSON, and potentially re-render, creating a massive O(N) performance bottleneck on every single state change. Additionally, passing inline objects/arrays (e.g. `[]`) as `initialValue` inside the dependency array of the event listener `useEffect` causes infinite listener re-attachment cycles on every render.
**Action:** Always dispatch `CustomEvent`s with targeted payloads (e.g., `{ detail: { key } }`) so hooks can filter events early and avoid unnecessary parsing/re-renders. Combine this with deep equality checks (`JSON.stringify(prev) === JSON.stringify(next)`) before updating React state, and omit non-primitive `initialValue`s from `useEffect` dependency arrays unless strictly required and stabilized.
